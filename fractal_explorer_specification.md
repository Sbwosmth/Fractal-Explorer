## 1. Загальний опис

**Назва проєкту:** Fractal Explorer (Дослідник Фракталів)

**Мета:** Інтерактивний додаток для візуалізації та дослідження фракталів, що демонструє використання концепції фракталів у комп'ютерній графіці.

**Цільова аудиторія:** Студенти, викладачі, ентузіасти математики та комп'ютерної графіки.

## 2. Технологічний стек

| Компонент | Технологія | Обґрунтування |
|-----------|------------|---------------|
| Мова програмування | Python 3.10+ | Вимога замовника |
| GUI фреймворк | PyQt6 | Сучасний, кросплатформний, багатий віджетами |
| Обчислення (CPU fallback) | NumPy + Numba | Швидкі векторні операції, JIT-компіляція |
| **GPU-прискорення** | **CuPy + CUDA** | Обчислення на відеокарті (RTX 3060), прискорення в 100-500 разів |
| Рендеринг | QImage + NumPy/CuPy | Пряма робота з піксельним буфером |
| Кольорові карти | Matplotlib colormaps | Готові професійні палітри |

### Чому CuPy, а не OpenCL?

| Критерій | CuPy (CUDA) | PyOpenCL |
|----------|-------------|----------|
| Для RTX 3060 | ✅ Нативна підтримка | ⚠️ Працює, але не оптимально |
| API | NumPy-сумісний | Низькорівневий |
| Кастомні ядра | `@cupy.fuse`, RawKernel | Писати на C |
| Швидкість розробки | Висока | Низька |
| Продуктивність | Максимальна для NVIDIA | Нижча |

CuPy дозволяє писати код майже як NumPy, але виконувати його на GPU. Для RTX 3060 — ідеальний вибір.

## 3. Функціональні вимоги

### 3.1. Типи фракталів

Додаток підтримує 4 типи фракталів:

#### 3.1.1. Множина Мандельброта
- **Формула:** zₙ₊₁ = zₙ² + c, де c — координата точки
- **Умова:** точка належить множині, якщо |z| ≤ 2 після N ітерацій
- **Колір:** визначається кількістю ітерацій до "втечі"

#### 3.1.2. Множина Жюліа
- **Формула:** zₙ₊₁ = zₙ² + c, де c — фіксована константа
- **Параметри:** дійсна та уявна частини константи c (налаштовуються користувачем)
- **Особливість:** форма кардинально змінюється при зміні c

#### 3.1.3. Фрактал Ньютона
- **Основа:** візуалізація методу Ньютона для рівняння z³ - 1 = 0
- **Формула:** zₙ₊₁ = zₙ - f(zₙ)/f'(zₙ) = zₙ - (zₙ³ - 1)/(3zₙ²)
- **Колір:** визначається тим, до якого кореня збіглася точка + кількість ітерацій

#### 3.1.4. Дерево Піфагора
- **Тип:** геометричний (L-система), не ітераційний
- **Побудова:** рекурсивне додавання квадратів на катетах трикутника
- **Параметри:** кут нахилу гілок (налаштовується)

### 3.2. Інтерактивне масштабування

| Дія користувача | Результат |
|-----------------|-----------|
| Колесо миші вгору | Збільшення (зум у 1.5x) відносно позиції курсора |
| Колесо миші вниз | Зменшення (зум у 0.67x) відносно позиції курсора |
| Затиснута ліва кнопка + рух | Панорамування (переміщення області перегляду) |
| Подвійний клік | Центрування на точці + збільшення |

### 3.3. Панель керування

#### Вибір фракталу
- Група з 4 кнопок (radio buttons)
- При перемиканні — автоматичне скидання до стандартного виду

#### Параметри ітерацій
- **Максимум ітерацій:** слайдер 50-2000, за замовчуванням 200
- Більше ітерацій = детальніше зображення, але повільніший рендеринг

#### Параметри множини Жюліа
- **Re(c):** слайдер від -2.0 до 2.0, крок 0.001
- **Im(c):** слайдер від -2.0 до 2.0, крок 0.001
- Показуються тільки коли вибрано фрактал Жюліа
- Пресети: "Кролик" (-0.123, 0.745), "Дракон" (-0.8, 0.156), "Спіраль" (-0.4, 0.6)

#### Параметри дерева Піфагора
- **Кут гілок:** слайдер 20°-70°, за замовчуванням 45°
- **Глибина рекурсії:** слайдер 5-15, за замовчуванням 10
- Показуються тільки коли вибрано дерево Піфагора

#### Кольорова схема
- Випадаючий список із палітрами: Twilight, Magma, Viridis, Plasma, Inferno, Turbo, Rainbow
- Попередній перегляд градієнта поруч із селектором

#### Кнопки дій
- **Скинути вид** — повернення до початкових координат та масштабу
- **Зберегти зображення** — експорт у PNG з поточними параметрами
- **Анімація** — запуск/зупинка автоматичної анімації параметрів

### 3.4. Інформаційна панель

Відображається внизу вікна:
- Поточні координати центру (Re, Im)
- Рівень масштабування (наприклад, "1.5×10⁶")
- Час останнього рендерингу (мс)
- Координати курсора в комплексній площині

### 3.5. Анімація

#### Режим анімації для Жюліа
- Параметр c рухається по колу в комплексній площині
- Радіус та швидкість налаштовуються
- Створює ефект "дихаючого" фракталу

#### Режим автозуму для Мандельброта
- Плавне наближення до заздалегідь визначеної цікавої точки
- Швидкість зуму налаштовується

## 6. Деталі реалізації

### 6.1. GPU Backend (CuPy + CUDA)

#### Автовизначення GPU
```python
class ComputeBackend:
    def __init__(self):
        self.use_gpu = False
        self.device_name = "CPU"
        try:
            import cupy as cp
            cp.cuda.Device(0).compute_capability  # Перевірка доступності
            self.use_gpu = True
            self.device_name = cp.cuda.Device(0).name
        except Exception:
            pass  # Fallback на CPU
```

#### CUDA-ядро для Мандельброта
```python
mandelbrot_kernel = cp.RawKernel(r'''
extern "C" __global__
void mandelbrot(float* output, int width, int height,
                double x_min, double x_max, double y_min, double y_max,
                int max_iter) {
    int px = blockIdx.x * blockDim.x + threadIdx.x;
    int py = blockIdx.y * blockDim.y + threadIdx.y;
    
    if (px >= width || py >= height) return;
    
    double x0 = x_min + (x_max - x_min) * px / width;
    double y0 = y_min + (y_max - y_min) * py / height;
    
    double x = 0.0, y = 0.0;
    int iter = 0;
    
    while (x*x + y*y <= 4.0 && iter < max_iter) {
        double xtemp = x*x - y*y + x0;
        y = 2*x*y + y0;
        x = xtemp;
        iter++;
    }
    
    // Smooth coloring
    float smooth_iter = iter;
    if (iter < max_iter) {
        float log_zn = logf(x*x + y*y) / 2.0f;
        float nu = logf(log_zn / logf(2.0f)) / logf(2.0f);
        smooth_iter = iter + 1 - nu;
    }
    
    output[py * width + px] = smooth_iter;
}
''', 'mandelbrot')
```

#### Виклик ядра
```python
def compute_mandelbrot_gpu(width, height, x_min, x_max, y_min, y_max, max_iter):
    output = cp.zeros((height, width), dtype=cp.float32)
    
    block = (16, 16)  # 256 потоків на блок
    grid = ((width + 15) // 16, (height + 15) // 16)
    
    mandelbrot_kernel(grid, block, (
        output, width, height,
        x_min, x_max, y_min, y_max, max_iter
    ))
    
    return output  # Залишається на GPU для подальшої обробки
```

### 6.2. CPU Fallback (Numba)

```python
@njit(parallel=True, fastmath=True)
def compute_mandelbrot_cpu(width, height, x_min, x_max, y_min, y_max, max_iter):
    result = np.empty((height, width), dtype=np.float32)
    for py in prange(height):
        for px in range(width):
            x0 = x_min + (x_max - x_min) * px / width
            y0 = y_min + (y_max - y_min) * py / height
            x, y = 0.0, 0.0
            iteration = 0
            while x*x + y*y <= 4.0 and iteration < max_iter:
                x_new = x*x - y*y + x0
                y = 2*x*y + y0
                x = x_new
                iteration += 1
            
            # Smooth coloring
            if iteration < max_iter:
                log_zn = np.log(x*x + y*y) / 2
                nu = np.log(log_zn / np.log(2)) / np.log(2)
                iteration = iteration + 1 - nu
            
            result[py, px] = iteration
    return result
```

### 6.3. Конвеєр рендерингу (GPU-оптимізований)

```
┌─────────────┐    ┌──────────────┐    ┌─────────────┐    ┌──────────┐
│ Viewport    │───▶│ GPU Kernel   │───▶│ Colormap    │───▶│ QImage   │

                         │                    │
                         └────────────────────┘
                           Все на GPU до останнього кроку
```

Кольорова карта застосовується на GPU через lookup table:
```python
def apply_colormap_gpu(iterations, max_iter, colormap_lut):
    # colormap_lut — cp.array shape (256, 4) на GPU
    normalized = (iterations / max_iter * 255).astype(cp.uint8)
    rgba = colormap_lut[normalized]  # Векторизований lookup на GPU
    return cp.asnumpy(rgba)  # Копіюємо в RAM тільки фінальне зображення
```

## 7. Користувацькі сценарії

### Сценарій 1: Перше знайомство
1. Користувач запускає додаток
2. Бачить множину Мандельброта в стандартному масштабі
3. Клікає на цікаву область на краю множини
4. Використовує колесо миші для наближення
5. Спостерігає, як з'являються нові деталі

### Сценарій 2: Дослідження Жюліа
1. Користувач вибирає "Множина Жюліа"
2. Обирає пресет "Дракон"
3. Плавно змінює параметр Re(c) слайдером
4. Бачить, як фрактал трансформується в реальному часі
5. Запускає анімацію для автоматичного "подорожі"

### Сценарій 3: Збереження результату
1. Користувач знаходить красиву область
2. Збільшує кількість ітерацій для максимальної деталізації
3. Обирає кольорову схему
4. Натискає "Зберегти зображення"
5. Отримує PNG файл високої якості
